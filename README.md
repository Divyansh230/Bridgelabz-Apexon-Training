### ðŸ”¹ Git Commands Learned & Implemented

- `git init`  
  Initialize a new Git repository.

- `git status`  
  Check the current state of the working directory and staging area.

- `git add <file>` / `git add .`  
  Move files from the working directory to the staging area.

- `git commit -m "message"`  
  Save staged changes permanently to the repository with a meaningful message.

- `git log`  
  View commit history along with commit IDs and messages.

- `git diff`  
  View changes between modified files and the last commit.

- `git restore --staged <file>`  
  Move a file back from the staging area to the working directory.

- `git checkout -- <file>`  
  Restore a file to its last committed state.

- `git clone <repository_url>`  
  Clone a remote repository to the local system.

- `git remote -v`  
  View connected remote repositories and their URLs.

- `git push`  
  Push local commits to the remote repository (GitHub).

- `.gitignore`  
  Used to exclude unnecessary files and folders from being tracked by Git.
